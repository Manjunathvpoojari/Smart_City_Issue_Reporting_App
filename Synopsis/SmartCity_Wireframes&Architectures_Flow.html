RLS Policy Design

Row Level Security enforced at database level — not just application level

R

Citizens — issues table

SELECT: user_id = auth.uid() (own issues only)
INSERT: always allowed when authenticated
UPDATE: not allowed (admin only)

A

Admins — issues table

SELECT: all rows (role = 'admin')
UPDATE: status + admin_note columns
DELETE: not in MVP scope

P

Public map view

SELECT on issues: all rows visible but only id, category, latitude, longitude, status exposed to unauthenticated public map queries

S

Storage bucket

issue-images bucket: authenticated upload, public read for CDN delivery. File path pattern: {user_id}/{issue_id}.jpg

Data Flow — 4 Core User Journeys

Step-by-step data movement for the main interactions in the app

🧑 Flow 1: Citizen Reports an Issue
1

Citizen opens app

Flutter checks Supabase Auth session. If expired, redirect to Google Sign-In screen.

2

Opens Report screen

Citizen taps FAB. Camera/gallery opens via image_picker. Photo selected or captured.

3

Location auto-tagged

geolocator fetches lat/lng from device GPS. Reverse-geocode via Nominatim (free) for address label.

4

Form submitted

Image uploaded to Supabase Storage → public URL returned. Issue row inserted to PostgreSQL with status = 'Pending'.

5

Realtime broadcast

PostgreSQL trigger fires on INSERT. Supabase Realtime broadcasts to all subscribed admin clients.

6

Success feedback

Flutter shows success snackbar. My Reports list refreshes via Riverpod provider re-fetch.

🏛️ Flow 2: Admin Updates Issue Status
1

Admin logs in

Admin enters credentials. Supabase Auth returns JWT with role = 'admin' in user metadata.

2

Dashboard loads

Riverpod fetches all issues (RLS allows admin SELECT all). Stats computed and displayed.

3

Issue selected

Admin taps issue. Full detail screen loads including citizen name, photo, GPS location.

4

Status changed

Admin selects new status from dropdown + adds note. Supabase UPDATE call fires with updated_at timestamp.

5

Push notification sent

Supabase Edge Function (or trigger) calls Firebase Admin SDK → FCM delivers notification to citizen device using stored fcm_token.

6

Citizen sees alert

Push notification received. App opens Issue Detail screen showing new status and admin note.

🗺️ Flow 3: Public Map Loads
1

Map screen opens

flutter_map renders OpenStreetMap tile layer (no API key needed, free CDN tiles).

2

Issues fetched

Supabase REST query: SELECT id, category, latitude, longitude, status FROM issues. Returns all public issues.

3

Pins rendered

Riverpod state maps issues to map markers. Color coded by status. Pending = red, Progress = blue, Resolved = green.

4

Live updates

Supabase Realtime subscription on issues table. New reports or status changes update map pins in real time via WebSocket.
